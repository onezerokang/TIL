# [Java] 어서와! 자료구조 알고리즘은 처음이지?/1일차

> 목표: 최대한 간단하게 줄일 것

## 오리엔테이션

- 프로그램은 정보를 처리하는 일을 한다.
  - 정보: 구조화된 데이터(자료구조)
  - 처리: 논리적 절차(알고리즘)
- 때문에 프로그램을 만들기 위해서는 자료구조와 알고리즘에 대한 이해가 필요하다.

## 시간복잡도

시간 복잡도란 입력되는 데이터가 증가할 때 성능의 변화를 예측하는 방법으로 주로 Big O 표기법을 사용한다.
Big O 표기법은 입력되는 데이터, n이 무한대로 증가할 때 수행해야 하는 처리가 얼만큼 증가해야 하는지에 대한 비율을 대략적으로 표기하는 방법이다.

- 자주 사용되는 Big O 표현식
  - `O(1)`: 데이터가 증가해도 처리 시간이 변하지 않는 경우
  - `O(n)`: 데이터가 n번 증가할 때 n번 더 수행하는 경우(증가 데이터에 비례하여 시간이 증가함)
  - `O(n^2)`: n이 증가할 수록 수행되는 연산은 n^2만큼 늘어나는 경우
  - `O(logn)`: 입력 데이터가 n개 일 때 연산이 수행될 때마다 처리횟수가 n개에서 절반씩 줄어들 때

## Array

Array는 메모리상에 연달아 공간을 확보하여 데이터를 저장하는 자료구조다.

<!-- 이미지 첨부할 것 -->

- Array의 사용 예시

```java
public class Main {
  public static void main(String[] args) {
    int[] arr = new int[5]; // 배열의 크기를 지정한다.
    System.out.println(arr); // 배열의 첫번째 요소의 주소가 출력된다.
    System.out.println(Arrays.toString(arr));//[0, 0, 0, 0, 0]

    arr[0] = 1; // index를 사용해서 값을 넣고 사용한다.
    System.out.println(arr[0]); // 1
    System.out.println(arr.length); // length로 배열의 크기를 알 수 있다.
  }
}
```

다음은 Array의 특징이다.

- 메모리상에 연달아 공간을 확보한다.
- 여러개의 데이터를 한꺼번에 다룰 수 있다.
- 첫 번째 요소의 위치만 알면 index로 상대적 위치를 빠르게 찾을 수 있다.
- 미리 공간을 확보해 놓고 써야 한다.
- 한 번 만들어진 공간은 크기가 고정된다.
- 미리 공간을 확보하고 크기가 변경되지 않기에 유연하지 못하다.
- Array는 Object는 아니지만 Reference Value로 취급된다.

## List

List는 Array랑 다르게 크기가 고정되어있지 않아 유연한 자료구조다.
처음부터 몇개의 공간이 필요할지 알 수 없기에 우선 하나만 만들어둔다.
이후 리스트에 데이터를 추가하면 메모리상 어딘가에 값을 저장하고
첫번째 요소가 두번째 요소를 찾을 수 있도록 링크를 연결하는 방식이다.

- List의 사용 예시

```java
import java.util.LinkedList;

public class Main {
  public static void main(String[] args) {
    LinkedList<Integer> list = new LinkedList<>();
    System.out.println(list); // [] toString이 오버라이딩 되어있음.
    list.add(1);
    list.add(2);
    list.add(3);
    System.out.println(list); // [1,2,3]

    list.add(1, 5);
    System.out.println(list); // [1,5,2,3]

    list.remove(2);
    System.out.println(list); // [1,5,3]

    System.out.println(list.get(2)) // 3
  }
}
```

다음은 알아두면 좋은 List의 메서드다.

- add()
- addAll()
- clear()
- get()
- set()
- indexOf()
- size()
- isEmpty()
- contains()

  - contains는 Object 타입을 받는다. 프리미티브 타입을 넣을 경우 자동으로 박싱돼서 Object타입으로 변환한다.
  - contains로 Object를 비교할 때 클래스에서 equals 메서드를 오버라이딩 해줘야 한다.

## List와 다형성

LinkedList는 List 인터페이스를 구현하고 있다. LinkedList 타입을 생성할 때 List 인터페이스 타입으로 표현하는 경우가 많다.
ArrayList, Vector, Stack도 List를 구현하고 있다.

```java
import java.util.List;
import java.util.LinkedList;
import java.util.Vector;

public class Main {
  public static void main(String[] args) {
    List<Integer> list = new Vector(); // Vector 형인데 리스트형으로도 표현가능하다. 벡터형이기도 하고 리스트형이기도 하다: 다형성
    List.add(1);
    List.add(2);
    method(List);
  }
  // ArrayList, LinkedList, Vector 모두 받을 수 있다.
  method(List<Integer> list) {
    System.out.println(list.isEmpty());
    System.out.println(list.size();)
  }
}
```

- linkedlist vs arraylist, vector
- linkedlist: double linked list 양방향으로 연결되어있는 구조.
- arraylist: resizable array arraylist안에 배열을 갖고 있고 필요에 따라서 그 크기를 늘린다.
  - 배열이 더 커져야 한다면 더 큰 배열을 만들고 기존 배열의 값을 복사한다. 이후 기존 배열을 제거한다.
  - 초기 배열의 크기는 10이다. arraylist를 생성할 때 초기화해줄 수 있다.
  - arrayList를 사용하면 get()으로 값을 찾아올 때 linkedlist보다 유리하다.
- vector: 내부에 array를 갖고 있다. arraylist와 비슷다. Vector is synchornized: 스레드 세이프한 작업이 필요할 때 사용한다. 추가 삭제 시 동기화 작업이 진행된다. 멀티 쓰레드 환경에서는 벡터를 사용해서 데이터의 정합성을 지켜주는 것이 중요하다면 벡터를 사용하는 것이 중요하다.

- 정리
  - LinkedList, ArrayList, Vector는 List 인터페이스를 구현한다.
  - LinkedList를 통해 double linked list를 사용할 수 있다.
  - ArrayList와 Vector는 둘다 내부에 array를 갖고 있다. 차이는 스레드 세이프
  - LinkedList Create, Update, Delete에 유리하고 Array는 인덱스로 값을 빠르게 접근하는데 유리하다.

Stream에 대해서도 공부를 해보면 좋을 것 같다.
코드를 더 간결하게 작성할 수 있음.

## List의 시간복잡도

|                      | add() | remove() | get() | contains() |
| -------------------- | ----- | -------- | ----- | ---------- |
| ArrayList            | O(1)  | O(n)     | O(1)  | O(n)       |
| LinkedList           | O(1)  | O(1)     | O(n)  | O(n)       |
| CopyOnWriteArrayList | O(n)  | O(n)     | O(1)  | O(n)       |

## 간편 리스트 만들기

- 고정 값으로 List 만들기

```
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
```

- Array로 List 만들기

```
Integer[] arr = new Integer[]{1, 2, 3, 4, 5};
List<Integer> list = Arrays.asList(arr);
```

- List로부터 List 만들기

```
List<Integer> list1 = Arrays.asList(1, 2, 3, 4, 5);
List<Integer> list2 = new LinkedList<>(list1);
```

주의: `Arrays.asList` 를 사용해서 간편하게 만들어낸 마치 배열처럼 고정된 크기의 리스트를 만들
어 냅니다. 이 리스트는 값을 변경할 수는 있지만, 크기에 영향을 주는 추가/삭제 작업은 할 수 없습니
다.

```
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
list.set(0, 6); // 가능
list.add(6); // error!
list.remove(0); // error!
```

## List를 Array로 변경하기

- for loop

```
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
int[] arr = new int[list.size()];
for (int i = 0; i < list.size(); i++) {
  arr[i] = list.get(i);
}
```

- toArray

```java
List<String> list = Arrays.asList("A", "B", "C", "D", "E");
String[] arr = list.toArray(String[]::new);
```

```java
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
Integer[] arr = list.toArray(Integer[]::new);
```

toArray를 사용할 때 만들려고 하는 배열타입의 생성방법을 지정해 줘야 한다.
하지만, 최종 배열타입이 다를 경우 사용할 수 없다.(auto unboxing이 적용되지 않는다)

- stream 사용하기

```
List<String> list = Arrays.asList("A", "B", "C", "D", "E");
String[] arr = list.stream().toArray(String[]::new);
```

stream을 통해서 toArray 를 사용하는 방법은 List의 toArray를 사용하는 것과 같습니다.
하지만, primitive 로 unboxing 이 필요한 경우 편리하게 사용될 수 있습니다.

```
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
int[] arr = list.stream().mapToInt(Integer::intValue).toArray();
```
