## CHAPTER 1 몽고DB 소개

### 1.1 손쉬운 사용

몽고DB는 관계형 데이터베이스가 아닌 도큐먼트 지향 데이터베이스다. 도큐먼트 지향 데이터베이스가 관계형 데이터베이스에 비해 갖는 이점은 다음과 같다.

1. 내장 도큐먼트와 배열을 허용하여 복잡한 계층관계를 하나의 레코드로 표현할 수 있다.
2. 고정된 스키마가 없어 필요할 때마다 쉽게 필드를 추가, 제거 할 수 있다.

### 1.3 다양한 기능

몽고DB는 범용 데이터베이스로 만들어졌기에 데이터의 CRUD와 데이터베이스 관리 시스템(DBMS)의 대부분의 기능과 더불어 다음과 같은 기능을 제공한다.

- 인덱싱

  - 몽고DB는 일반적인 보조 인덱스를 지원하며 고유(unique), 복합(compound), 공간 정보, 전문(full-text) 인덱싱 기능도 제공한다. 중첩된 도큐먼트 및 배열과 같은 계층 구조의 보조 인덱스도 지원하며 개발자는 모델링 기능을 자신의 애플리케이션에 가장 적합한 방식으로 최대한 활용할 수 있다.

- 집계

  - 몽고DB는 데이터 처리 파이프라인 개념을 기반으로 한 집계 프레임워크를 제공한다. 집계 파이프라인(aggregation pipeline)은 데이터베이스 최적화를 최대한 활용해, 서버 측에서 비교적 간단한 일련의 단계로 데이터를 처리함으로써 복잡한 분석 엔진(analytics engine)을 구축하게 해준다.

- 특수한 컬렉션 유형

  - 몽고DB는 로그와 같은 최신 데이터를 유지하고자 하는 세션이나 고정 크기 컬렉션(제한 컬렉션 capped collection)과 같이 특정 시간에 만료해야 하는 데이터에 대해 TTL(Time-to-Live) 컬렉션을 지원한다. 또한 기준 필터와 일치하는 도큐먼트에 한정된 부분 인덱스를 지원함으로써 효율성을 높이고 필요한 저장 공간을 줄인다.

- 파일 스토리지

  - 몽고DB는 큰 파일과 파일 메타데이터를 편리하게 저장하는 프로토콜을 지원한다.

  관계형 데이터베이스의 Join은 몽고DB에 존재하지 않는다.
