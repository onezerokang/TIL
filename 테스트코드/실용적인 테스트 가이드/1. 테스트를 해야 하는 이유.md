# 테스트를 해야 하는 이유

## 1. 주니어 개발자가 테스트를 학습 해야 하는 이유

테스트를 작성하는 역량은 중요하다.

- 주니어 개발자에게 가장 기대하는 요소 중 하나다.
- 채용 시 구현 과제 등에서 테스트 작성 여부, 테스트 코드 구현 방식을 확인한다.
- 소프트웨어 품질을 보증하는 방법으로, 그 중요성을 알고 있는지를 확인할 수 있다.

## 2. 테스트를 해야 하는 이유

테스트 코드를 작성하는 것을 귀찮게 느낄 수 있다. 따라서 왜 테스트를 해야 하는지에 대해 알지 못하면 테스트 작성에 대한 동기가 떨어지고 작성하지 않게 될 것이다.

### 2.1. 수동 테스트의 한계

테스트 코드를 작성하지 않고 기능이 추가될 때마다 수작업으로 하나씩 눌러보며 테스트 하는 사람(혹은 사람)이 있다(필자도 과거 회사에서 그랬다).

이렇게 수동으로 테스트하면 다음과 같은 문제가 발생한다.

- 커버할 수 없는 영역 발생
- 경험과 감에 의존
- 늦은 피드백
- 유지보수 어려움
- 소프트웨어 신뢰 ↓

새로운 기능이 추가될 때마다 테스트를 해야 하고, 추가된 기능을 수정하며 기존 기능이 같이 수정 되었다면 변경된 모든 부분을 전부 수동으로 테스트 해야 한다.

### 2.2. 안 좋은 테스트 코드의 한계

수동 테스트를 줄이기 위해 테스트 코드를 작성했으나, 안 좋은 코드를 작성하면 다음과 같은 문제가 발생한다.

- 프로덕션 코드의 안정성을 제공하기 힘들어진다.
- 테스트 코드 자체가 유지보수하기 어려운, 새로운 짐이 된다.
- 잘못된 검증이 이루어질 가능성이 생긴다.

### 2.3. 테스트 코드의 장점

테스트 코드를 이해하고, 잘 작성한다면 다음과 같은 장점을 얻을 수 있다.

- 자동화 테스트로 버그를 조기 발견하고, 수동 테스트 비용을 크게 절약
- 소프트웨어의 빠른 변화를 지원
- 팀원들의 집단 지성을 팀 차원의 이익으로 승격
- 가까이 보면 느리지만, 멀리 보면 가장 빠르다.
