# 단위 테스트

단위 테스트란 **작은** 코드 단위를 **독립적**으로 검증하는 테스트로 검증 속도가 빠르고 안정적이라는 특징을 갖는다.

여기서 "작은"의 의미는 클래스나 메소드 단위를 의미하고, "독립적"이라는 의미는 네트워크나 DB 같이 외부에 의존하지 않음을 의미한다.

자바에서는 JUnit5 프레임워크와 AssertJ 라이브러리를 사용하여 단위 테스트를 자동화 한다.

## 1. 테스트 케이스 세분화하기

요구사항이 주어지고 테스트 케이스를 작성할 때 해피 케이스와 예외 케이스에 대해 생각해봐야 한다.

- 해피 케이스
  - 예상대로 잘 동작하는지 확인하기 위한 테스트
  - 예시: SNS에서 회원 정보를 입력하고 회원 가입이 완료되는 과정을 테스트 하는 경우
- 예외 케이스
  - 비정상적이거나 예외적인 상황에서 시스템이 어떻게 반응하는지 확인하기 위한 테스트
  - 예시: SNS에서 이미 등록된 이메일을 입력하고 회원 가입을 하려고 할 때 처리를 테스트

위 케이스에 대한 테스트 코드를 작성할 때 **경계값** 테스트를 해야 한다(정말 중요하다).

경계값 테스트란 테스트 케이스를 설계하는 기법 중 하나로, 입력값의 경계에서 문제가 발생하는 경우가 많기에, 해당 경계를 중심으로 테스트를 수행하는 방법이다.

예를들어 회원 가입 시 나이를 입력할 때 허용 범위를 0 초과 100미만으로 지정했을 때 0과 1 그리고 99와 100에서 테스트를 진행해보는 것이다.

## 2. 테스트하기 어려운 영역 분리하기

테스트 하기 어려운 코드 영역을 구분하고 외부로 분리해서 테스트 하기 좋은 코드로 만들어야 한다. 다음은 테스트 하기 어려운 코드의 특징과 쉬운 코드의 특징이다.

- 테스트하기 어려운 영역

  - 관측할 때마다 다른 값에 의존하는 코드(Input)
    - 현재 날짜/시간, 랜덤 값, 전역 변수/함수, 사용자 입력
  - 외부 세계에 영향을 주는 코드(Output)
    - 표준 출력, 메시지 발송, 데이터베이스에 기록하기

- 순수 함수
  - 같은 입력에는 항상 같은 결과
  - 외부 세상과 단절된 형태
  - 테스트하기 쉬운 코드

## 3. 키워드 정리

- 핵심 키워드
  - 단위 테스트
  - 수동 테스트, 자동화 테스트
  - Junit5, AssertJ
  - 해피 케이스, 예외 케이스
  - 경계값 테스트
  - 테스트하기 쉬운/어려운 영역 (순수함수)
- 추가 키워드
  - lombok(사용 가이드)
    - @Data, @Setter, @AllArgsConstructor 지양
    - 양방향 연관 관계 시 @ToString 순환 참조 문제(DTO에서만 사용)
