# HTTP

## 1. 개요

HTTP(Hyper Text Transfer Protocol)은 인터넷에서 클라이언트와 서버가 데이터를 주고 받기 위한 프로토콜이다.
주로 TCP를 사용하고 HTTP/3부터는 UDP를 사용하며 80번 포트를 사용한다.

HTTP의 대표적인 특징으로는 '**무상태성**'와 '**비연결성**'이 있다.

## 2. 무상태성(Stateless)

무상태(Stateless)란 서버가 클라이언트의 상태를 보관하지 않는 것을 의미한다.

- **장점**:
  - **서버 확장성**: 서버가 클라이언트의 상태를 보관하지 않기 때문에 수평적 확장(Scale Out)에 유리하다.
- **단점**:
  - **상태 보관**: 요청할 때마다 클라이언트는 자신이 누구인지, 어떤 작업을 진행중이였는지에 대한 데이터를 추가로 전송해야 한다.

## 3. 비연결성(Connectionless)

비연결성(Connectionless)란 요청과 응답을 주고받은 후 통신이 끊기는 것을 의미한다.

- **장점**:
  - **서버 리소스 절약**: 요청과 응답을 주고 받으면 통신을 끊기 떄문에, 통신이 끝난 후에 서버의 리소스가 소모되지 않는다.
- **단점**:
  - **반복되는 TCP 핸드셰이크**: 웹 페이지 로드 같이 한번에 여러개의 자원을 요청할 경우, 각 자원마다 TCP 핸드셰이크를 진행해야 한다.

### 3.1. 지속 연결(Persistent Connection)

지속 연결은 비 연결성의 한계를 극복하기 위해 HTTP/1.1에 추가된 방식이다.

웹 페이지를 로드할 경우 HTML뿐만 아니라 CSS, JavaScript, 이미지 등 여러 자원을 요청하게 되는데 자원을 요청할 때마다 TCP 핸드셰이크를 진행할 경우 보다 많은 시간이 소요되었다.

웹 초기에는 사이트 콘텐츠가 많지 않아 이에 대한 부담이 크지 않았으나 이미지 등의 멀티미디어 콘텐츠가 늘어나면서 TCP 연결의 재사용이 필요하게 되었다.

HTTP/1.0에서 지속 연결을 하기 위해서는 요청 헤더에 다음을 추가하면 된다.

```
Connection: keep-alive
```

HTTP/1.1부터는 Connection 헤더를 사용하지 않더라도, 기본적으로 Persistent Connection을 지원한다.
