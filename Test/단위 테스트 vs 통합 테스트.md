# 단위 테스트 vs 통합 테스트

## 테스트 코드란

테스트 코드란 프로그램이 의도한대로(예상한대로) 동작하는지 테스트하는 코드를 말한다. 예상하지 못한 결과가 발생할 경우 테스트에 실패하기 때문에 문제되는 부분을 빠르게 파악할 수 있다.

테스트 코드를 작성하는 것의 장점은 다음과 같다.

- **소프트웨어 안정성 보장**: 기능이 추가되거나, 코드를 리팩토링하여 문제가 발생했을 때 즉각적인 피드백을 받을 수 있다. 이를 통해 소프트웨어 안정성을 보장한다.
- **프로덕션 코드 이해 도움**: 테스트 코드와 테스트 케이스를 읽는 것만으로 프로덕션 코드를 이해하는데 도움이 된다.

## 단위 테스트(Unit Test)

단위 테스트는 작은 코드 단위(클래스 or 메소드)를 독립적으로 검증하는 테스트다.

외부에 의존하지 않는 독립적이고 작은 단위로 테스트하기 때문에 테스트 속도가 빠르다는 장점이 있다.

Java의 경우 JUnit 테스트 프레임워크를 통해 단위 테스트를 할 수 있다.

다음은 JUnit5와 AssertJ를 이용해 계산기 클래스의 더하기 메소드를 테스트하는 예시 코드다.

```java
// Calculator.java
public class Calculator {
  public int add(int a, int b) {
    return a + b;
  }
}


// CalculatorTest.java
class CalculatorTest {
  @DisplayName("두 개의 값을 더한다")
  @Test
  void add() {
    Calculator calc = new Calculator();
    int a = 3;
    int b = 5;

    int result = calc.add(3, 5);

    assertThat(result).isEqualTo(8);
  }
}
```

## 통합 테스트(Integration Test)

통합 테스트는 여러 모듈이 협력하는 기능을 통합적으로 검증하는 테스트다.

단위 테스트가 모두 통과한다고 하더라도 기능 전체의 신뢰성을 보장할 수 없다. 각 메소드가 예상한대로 잘 동작한다고 해도, 이들이 협력하는 과정이나, 외부 요인(DB 연결, 외부 라이브러리 등)으로 문제가 발생할 수 있기 때문이다.

때문에 통합 테스트는 DB에 접근하거나 전체코드와 다양한 환경이 제대로 작동하는지 필요한 모든 작업을 수행할 수 있다.

## 비교

| 단위 테스트                        | 통합 테스트                                                               |
| ---------------------------------- | ------------------------------------------------------------------------- |
| 결과는 자바 코드에만 영향을 받는다 | 결과는 외부 시스템에 영향을 받을 수 있다                                  |
| 비교적 작성하기 쉽다               | 구축하기 꽤 까다롭다                                                      |
| 각 단위를 독립적으로 테스트한다    | 하나 이상의 구성 요소를 테스트한다                                        |
| 모킹으로 의존성을 배제할 수 있다   | 모킹 사용을 권장하지 않는다                                               |
| 구현한 코드만 테스트한다           | 구성 요소의 구현을 테스트하고, 구성 요소 간의 상호 연결 동작을 테스트한다 |
| 테스트 실행 시간이 길지 않다       | 테스트 실행 시간이 꽤 길다                                                |
