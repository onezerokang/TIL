# 단위 테스트 vs 통합 테스트

## 테스트 코드란

테스트 코드란 프로그램이 의도한대로(예상한대로) 동작하는지 테스트하는 코드를 말한다.

테스트 코드를 작성하는 것의 장점은 다음과 같다.

1. 소프트웨어의 안정성을 보장한다: 기능이 추가되거나, 코드를 리팩토링했을 때 테스트가 통과함으로써
2. 테스트 자체가 문서가 된다:
3. 결합도가

## 단위 테스트(Unit Test)

단위 테스트는 작은 코드 단위(클래스 or 메소드)를 독립적으로 검증하는 테스트다.

외부에 의존하지 않는 독립적이고 작은 단위로 테스트하기 때문에 테스트 속도가 빠르다는 장점이 있다.

Java의 경우 JUnit 테스트 프레임워크를 통해 단위 테스트를 할 수 있다.

## 통합 테스트(Integration Test)

통합 테스트는 여러 모듈이 협력하는 기능을 통합적으로 검증하는 테스트다.

단위 테스트는 기능 전체의 신뢰성을 보장할 수 없다.

각 메소드가 예상한대로 잘 동작한다고 해도, 이들이 협력하는 과정에서 문제가 발생할 수 있기 떄문이다.
