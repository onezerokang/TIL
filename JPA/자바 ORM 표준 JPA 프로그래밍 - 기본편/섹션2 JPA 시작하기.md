## Hello JPA - 애플리케이션 개발

### JPA 구동 방식

1. JPA는 Persistence라는 클래스가 있다. 이 클래스가 META-INF/persistence.xml이라는 설정 정보를 조회한다.
2. 설정정보를 바탕으로 EntityManagerFactory를 만든다.
3. 필요할 때마다 EntityManger를 만들어 사용한다.

수정할 때는 저장 메서드를 따로 호출할 필요가 없다. 자바 컬렉션 처럼 사용할 수 있기 떄문에
JPA를 통해 엔티티를 가져오면 JPA가 가져온 것을 관리한다. 그리고 트랜잭션이 커밋되는 시점에 변경사항을 체크하고 업데이트 쿼리를 날린다.

엔티티 매니저 팩토리닌 서버가 뜰 때 한번만 생성되고
엔티티 매니저는 고객의 요청이 올 때 생성하고 처리후 닫는다.
엔티티 메니저는 쓰레드간에 공유하면 안된다(사용하고 버려야 한다.)
**JPA의 모든 데이터변경은 트랜잭션 안에서 실행돼야 한다.**

DB는 내부적으로 트랜잭션 개념을 다 가지고 있다.

### JPQL

- 가장 단순한 조회 방법: em.find()
- 복잡한 쿼리를 할 때는 JPQL을 사용해야 한다.

// jpa는 테이블을 대상으로 쿼리하지 않는다.
// 멤버 객체를 가져오는 jpql
// 객체를 대상으로 하는 객체 지향 쿼리 -> db에 맞게 번역을 해준다.

- JPA를 사용하면 엔티티 객체 중심으로 개발
- 문제는 검색 쿼리
- 검색을 할 때도 테이블이 아닌 엔티티 객체를 대상으로 검색
- 모든 DB 데이터를 객체로 변환해서 검색하는 것은 불가능
- 애플리케이션이 필요한 데이터만 DB에서 불러오려면 결국 검색 조건이 포함된 SQL이 필요

테이블에서 가져오면 jpa의 사상이 깨지기 때문에 객체를 대상으로 쿼리를 한다.

- JPA는 SQL을 추상화한 JPQL이라는 객체 지향 쿼리 언어 제공
- SQL문법과 유사, SELECT, FROM, WHERE, GROUP BY, HAVING, JOIN 지원
- JPQL은 엔티티 객체를 대상으로 쿼리
- SQL은 데이터베이스 테이블을 대상으로 쿼리
