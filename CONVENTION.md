# TIL 컨벤션 가이드

## 폴더 운영

- 기술/주제 단위로 **1depth만 유지**한다.
- 빈 폴더를 미리 만들지 않는다. 첫 글을 쓸 때 생성한다.
- 폴더는 **10~15개 이내**로 유지한다. 글이 적은 주제는 `etc/`에 넣다가 나중에 분리한다.
- 폴더 정렬은 알파벳 순 자동 정렬을 그대로 사용한다.

## 파일 이름

- **영문 케밥 케이스**를 사용한다. 한글, 쉼표, 공백, 괄호, 특수문자는 사용하지 않는다.
- 파일명에 날짜를 넣지 않는다. git log가 이력 역할을 한다.
- 클래스명도 케밥 케이스로 변환한다. (`HashMap` → `hash-map.md`)
- 파일 안 제목(`# 제목`)은 자유롭게 쓴다.

## 언어 & 데이터 접근

- Java와 Kotlin은 **별도 폴더**로 관리한다.
- `jpa/`를 기본으로 유지하고, JDBC 글이 생기면 그때 `jdbc/`를 만든다.

## 이미지

- 각 주제 폴더 안에 `images/` 하위 폴더를 둔다.
- 이미지 파일이름도 파일 이름 규칙을 따른다.
- 마크다운에서 상대경로로 참조한다.

## 책 / 강의 정리

- `books/`, `courses/`에 **책(강의) 단위 폴더**로 둔다.
- 각 폴더 안에 `README.md`를 두고 목차를 정리할 수 있다.

## 루트 README

- Books / Courses는 테이블로 진행률과 링크를 정리한다.
- Daily TIL은 **월별 역순**으로 링크를 정리한다.

## 커밋 메시지

카테고리를 prefix로 사용한다. (`jpa: n+1 문제 해결`)

## 작성 도구

IntelliJ에서 바로 작성한다. TIL 레포를 프로젝트로 열어두고 탭 전환으로 작성한다.
