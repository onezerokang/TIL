# 프로세스와 스레드

프로그램: 컴퓨터가 실행할 수 있는 명령어들의 집합
프로세스: 컴퓨터에서 실행 중인 프로그램. 각각의 프로세스는 독립된 메모리 공간을 할당 받는다. 메모리 공간에 명령어와 데이터를 갖게 된다.

CPU: 명령어를 실행하는 연산 장치

메인 메모리: 프로세스가 CPU에서 실행되기 위해 대기하는 곳

IO: 파일을 읽고 쓰거나 네트워크 어딘가와 데이터를 주고 받는것, 입출력 장치와 데이터를 주고 받는 것

## 단일 프로세스 시스템

한 번에 하나의 프로그램만 실행됨

하나의 프로그램이 실행 중이면 또 다른 프로그램을 실행할 수 없었다.

CPU 사용률이 안 좋다는 단점이 있다.

IO 작업을 하는 중에는 CPU를 사용하지 않는다. IO 작업을 하는 동안 CPU는 놀게 된다. IO 작업을 하는 동안 다른 프로세스가 CPU를 점유하면 더 효율적일 것

여러 개의 프로그램을 메모리에 올려놓고 동시에 실행시키자. IO 작업이 발생하면 다른 프로세스가 CPU에서 실행된다.

## 멀티프로그래밍

CPU 사용률을 극대화 시키기 위한 목적

이도 단점이 있었는데 CPU 사용률이 길어지면 다른 프로세스는 계속 대기해야 한다.

해결책: 프로세스는 한번 CPU를 사용할 때 아주 짧은 시간(quantum) CPU에서 실행되도록 하자

## 멀티태스킹

멀티프로그래밍과 유사하지만 CPU 타임을 아주 짧게 쪼개서 프로세스가 번갈아가며 실행되도록 한 것.

프로세스의 응답시간을 최소화 시키는데 목적이 있다. 일반 사용자가 느끼기에 여러 프로그램이 동시에 실행되는 것처럼 느끼게 된다.

단점: 하나의 프로세스가 동시에 여러 작업을 수행하지 못한다. 물론 여러 프로세스를 실행시키면 되지만 단점 있음

1. 프로세스의 컨텍스트 스위칭은 무거운 작업
2. 프로세스끼리 데이터 공유가 까다롭다(독립적인 메모리 공간을 점유하고 있기 때문)
3. 듀얼 코어가 등장했는데 잘 쓰고 싶음

## 스레드

프로세스는 한 개 이상의 스레드를 가질 수 있다.

CPU에서 실행되는 단위(unit of execution)

현대 컴퓨터에서 스레드는 프로세스의 실행 단위이다.

같은 프로세스의 스레드들끼리 컨텍스트 스위칭은 가볍다.

스레드들은 자신들이 속한 프로세스의 메모리 영역을 공유한다. 데이터 공유가 쉽다.

멀티스레드는 heap 영역을 공유한다.

하지만 각 스레드의 스택은 공유하지 않는다.

프로그램 카운터: 다음번에 실행될 명령어의 메모리 주소를 가르키는 레지스터

## 멀티스레딩

하나의 프로세스가 동시에 여러 작업을 실행하는데 목적

확장된 멀티태스킹 개념: 여러 프로세스와 여러 스레드가 아주 짧게 쪼개진 cpu 타임을 나눠 갖는 것

## 멀티프로세싱

듀얼코어 CPU에서 복수의 프로세스를 돌림

## 예제

싱글코어 CPU에 싱글-스레드 프로세스 두 개 -> 멀티태스킹

싱글코어 CPU에 듀얼-스레드 프로세스 한 개 -> 멀티스레딩

듀얼코어 CPU에 싱글-스레드 프로세스 두 개 -> 멀티프로세싱

듀얼코어 CPU에 듀얼-스레드 프로세스 한 개 -> 멀티스레드, 멀티프로세싱

듀얼코어 CPU에 듀얼-스레드 프로세스 두개 -> 멀티태스킹, 멀티스레드, 멀티프로세싱
