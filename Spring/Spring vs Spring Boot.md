# Spring vs Spring Boot

## Spring 생태계

스프링은 하나의 기술이 아닌 여러 스프링 프로젝트들의 모음이다. 모든 스프링 프로젝트는 스프링 프레임워크를 기반으로 동작한다.

스프링 생태계가 커지고, 스프링 프레임워크가 복잡해지며 스프링을 사용하기 어려줘졌다. 이를 해결하기 위한 프로젝트가 스프링 부트다.

## Spring Framework

- 자바 기반의 어플리케이션 개발
- 개발자는 비즈니스 로직에 집중

스프링 이전에 EJB 프레임워크가 있었다. 비즈니스 로직과 앤터프라이즈 기술의 분리를 목적으로 탄생했다.

- 실제: 너무 비쌈
- 비즈니스 로직보다 EJB를 사용하기 위한 코드가 너무 많음
- EJB에 너무 의존적이고 종속적인 개발 -> 자바 장점 못살림

이를 해결하기 위한 것이 스프링

스프링 생태계 커지고, 스프링 프레임워크 많아지고, 다양한 오픈소스 등장 -> 외부 라이브러리 함께 사용할 일이 많아졌다.

스프링으로 개발을 시작할 때 직접 해야 하는 설정이 너무 많아지고 복잡해짐.

## Spring Boot

스프링 부트를 사용하면 스프링 기반의 애플리케이션을 쉽게 만들 수 있다.

스프링 개발에 필요한 복잡한 설정은 Spring Boot가 해준다.

개발자는 즉시 어플리케이션 개발 시작 가능.

## Spring Boot 덕붕네 우리는!

1. 내장 서버
   - 스프링 부트가 없던 시절, 웹 어플리케이션 배포하려면
     - WAS 설치
     - 개발한 웹 애플리케이션을 WAR로 빌드
     - 빌드한 WAR 파일을 WAS 폴더에 넣고
     - WAS 실행
   - 스프링부트는 톰캣을 라이브러리로 포함하는 내장 톰캣을 제공한다.
   - WAR가 아닌 JAR 파일로 패키징
   - 즉 main 메소드로 바로 실행 가능.
2. 편리한 의존성 & 권장 버전 관리
   - 어떤 라이브러리를 다 고르고, 각 라이브러리의 버전도 골라줘야 한다.
   - 버전이 서로 호환되지 않는 경우도 있다.
   - 스프링부트는 라이브러리 버전 관리를 돕는다.
     - spring-dependency-managment 설치하면 버전 생략 가능
     - 기본 라이브러리 묶음을 제공한다. spring-boot-starter
3. 자동 구성
   - 일반적으로 자주 사용하는 수 많은 빈들을 자동으로 등록해주는 기능
   - JdbcTemplate, DataSource, TransactionManager
   - spring-boot-autoconfigure 모듈을 통해 수 많은 자동 구성을 제공한다.
