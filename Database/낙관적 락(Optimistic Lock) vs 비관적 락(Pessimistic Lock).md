# 낙관적 락(Optimistic Lock) vs 비관적 락(Pessimistic Lock)

## 개요

Database Lock은 동시성을 보장하기 위한 기능이다.

Lock의 종류로는 여러 트랜잭션 간 충돌이 일어나지 않을 것이라 가정하는 낙관적 락(Optimistic Lock)과 여러 트랜잭션 간 충돌이 자주 일어날 것이라 가정하는 비관적 락(Pessimistic Lock)이 있다.

## 낙관적 락(Optimistic Lock)

실제로 락을 걸지 않고 버전을 이용해 정합성을 맞추는 방식이다.

처리하고 있는 데이터의 버전이 변경될 경우, 실패처리시킨다.

낙관적 락은 버전 불일치 시 처리를 애플리케이션 레벨에서 담당한다. 즉, 충돌이 발생하여 실패할 경우, 재시도하는 로직을 개발자가 구현해야 한다.

별도의 락을 잡지 않아, 비관적 락보다 성능상 이점이 있지만, 충돌이 자주 발생한다면 어플리케이션 레벨에서 다시 자원에 접근해야 하기 때문에 별로임.

## 비관적 락(Pessimistic Lock)

실제 데이터에 락을 걸어 데이터 정합성을 지키는 방식이다.

하나의 트랜잭션이 자원에 접근해 작업을 처리하고 있다면, 다른 트랜잭션은 이를 대기해야 한다.

별도의 락을 걸기 때문에 낙관적 락에 비해 성능이 좋지 않다.
