# index란?

## 1. 개요

인덱스는 데이터베이스에서 데이터 검색 속도를 향상 시키기 위한 자료구조다. 특정 컬럼에 대한 인덱스를 생성하면, 해당 컬럼을 기반으로 한 쿼리가 실행될 때 인덱스를 통해 데이터를 빠르게 검색하고, 해당하는 레코드를 반환한다.

인덱스를 잘 활용하면 조회 성능을 높일 수 있지만, 인덱스 자체가 저장 공간을 차지하고, CUD(생성, 수정, 삭제) 작업 시 인덱스 업데이트가 필요하기 때문에, 너무 많은 인덱스가 있으면 오히려 성능이 떨어질 수 있다.

## 2. 인덱스 자료구조

### 2.1. B-tree

대부분의 인덱스는 B-Tree(Balanced Tree)를 사용한다. B-Tree는 루트 노드, 브랜치 노드(중간 노드), 리프 노드로 구성된 자료구조다.

B-tree는 자식 노드의 개수가 2개 이상인 균형 검색 트리(Binary Search Tree)다. 각 노드는 키의 범위를 가지며, 이 범위는 해당 노드의 모든 하위 키를 포함한다. 따라서 노드를 이동할 때마다 탐색 범위를 2/3 ~ 3/4씩 줄여가며 데이터를 찾을 수 있다.

하지만 B-Tree는 삽입/수정/삭제가 발생할 경우 리밸런싱 작업이 발생하기에, 남발할 경우 성능 저하의 원인이 된다.

### 2.2. B+Tree

B+tree를 사용해서 인덱스를 구성하는 DB도 있는데 부모 노드는 데이터 탐색 가이드를 제공하고, 리프 노드에만 데이터를 보관하는 트리다. 리프 노드끼리도 연결되어 있다. 범위 검색이 매우 쉬워진다.

### 2.3. Hash

Hash 인덱스는 특정 컬럼 값을 해싱하여 인덱싱하여 값을 빠르게 찾는다. 이런 특징으로 인해 동등 비교 검색 시 조회 속도가 매우 빠르다. 또한 컬럼의 값이 아무리 길어도 해싱된 값은 일정하기에 4 ~ 8바이트 수준으로 상당히 줄어드는 장점이 있다.

하지만 값이 정렬되어 있진 않기에 범위 조회와 정렬 조회의 성능은 느리고, 해시 충돌이 발생할 경우, 여러 데이터가 동일한 해시 값을 갖게 되어 성능 저하가 발생할 수 있다.

## 3. 인덱스를 만드는 방법

인덱스를 만드는 방법은 데이터베이스마다 다르다. 본 게시글에서는 MySQL과 MongoDB를 예시로 설명한다.

### 3.1. MySQL

- **클러스트형 인덱스**: 테이블당 하나를 설정할 수 있다. PK 옵션으로 기본키를 만들면 클러스터형 인덱스를 생성할 수 있고, 기본키를 만들지 않고 unique not null 옵션을 붙이면 만들 수 있다.
- **세컨더리 인덱스**: create index 명령어를 기반으로 만들 수 있다.

하나의 인덱스만 생성할 것이라면 클러스터형 인덱스를 만드는 것이 세컨더리 인덱스를 만드는 것보다 효율이 좋다.

세컨더리 인덱스는 보조 인덱스로 여러 개의 필드 값을 기반으로 쿼리를 많이 보낼 때 사용한다.

### 3.2. MongoDB

도큐먼트를 만들면 자동으로 ObjectID가 형성되며, 해당 키가 기본 키가 된다. 세컨더리 키도 부가적으로 설정할 수 있다.

## 4. 인덱스 최적화 기법

인덱스를 사용할 경우 인덱스 저장 시 필요한 용량이 추가로 들고, 데이터에 변화가 생겼을 경우 인덱스도 적용되어야 하기 때문에 필요한 곳에만 사용해야 한다.

다음은 인덱스를 최적화 하는 방법이다.

- **테스팅**: explain() 함수를 통해 인덱스를 만들고 쿼리를 보낸 이후에 테스팅을 하며 걸리는 시간을 최소화 한다.
- **인덱스 생성 우선순위**: 여러 필드를 기반으로 데이터를 조회할 시 복합 인덱스를 생성해야 한다. 인덱스 생성 순서에 따라 인덱스 성능이 달라진다.
  1. 같음을 비교하는 ==이나 equal 쿼리가 있다면 제일 먼저 인덱스로 설정한다.
  2. 정렬에 쓰는 필드라면 그다음 인덱스로 설정한다.
  3. 다중 값을 출력해야 하는 필드, 즉 쿼리 자체가 > 이거나 < 등 많은 값을 출력해야 하는 쿼리에 쓰는 필드라면 나중에 인덱스를 설정한다.
  4. 유니크한 값의 정도를 카디널리티라고 한다. 높은 순서대로 인덱스를 생허애햐 한다. email과 age 중 email의 유니크함이 더 높기 때문에 email 필드에 대한 인덱스를 먼저 설정해야 한다.

```SQL
EXPLAIN
SELECT * FROM t1
JOIN t2 ON t1.c1 = t2.c1
```
