# Node.js란

Node.js는 Chrome v8 engine으로 빌드된 자바스크립트 런타입이다.
여기서 자바스크립트 런타임이란 자바스크립트가 실행되는 환경을 의미한다.
node.js를 설치하고 터미널에 `node [파일명]`을 입력하면 JS파일을 실행하는 것을 볼 수 있다.

그래서 이런 JS 런타임을 사용하여 서버를 구축할 수도 있고, 크롤러를 만들 수도 있는 것이다.

Node.js는 크게 3가지의 특징을 가지고 있다.

1. Event Driven
2. Single Thread
3. Non-blocking & Asynchronous

## Event Driven

이벤트 기반이란 클릭, 네트워크 요청 같은 이벤트가 발생할 때 미리 지정해둔 작업을 수행하는 방식이다.
보통 이벤트 리스터에 콜백함수를 등록하여 이벤트가 발생하면 등록한 콜백 함수를 실행한다.

## Process

스레드를 알기 위해서는 프로세스가 무엇인지 알아야 한다.
프로세스는 프로그램이 운영체제에서 메모리 공간을 할당받아 실행중인 프로그램이다.
프로세스에는 프로그램에 사용되는 데이터와 메모리 등의 자원 그리고 **스레스**로 구성된다.

## Thread

스레드는 프로세스 내에서 작업을 하는 주체다.
여기서 프로세스 내 스레드가 한개면 싱글 스레드 두개 이상이면 멀티 스레드라고 한다.

## Single Thread

Node.js에서 개발자가 제어할 수 있는 스레드는 하나이다. 그래서 노드는 싱글 스레드로 여겨진다.
백그라운드에서 스레드 풀을 구성하여 File System이나 HTTP Request, setTimeout 같은 작업이 필요하면
백그라운드에 넘겨 처리한다(Node.js에 포함된 libuv가 그 역할을 한다.)
개발자는 여러 스레드를 관리하지 않고 한개의 스레드만을 사용하면 되기 때문에 코딩의 복잡도가 감소한다.

## Non-blocking & Asynchronous

위에도 작성하였지만 Node.js는 싱글 스레드를 사용하기 때문에 한번에 하나의 요청만을 처리한다.
그래서 블로킹이 발생할 것 같은 경우에는 논 블로킹 방식으로 대기 시간을 최대한 줄인다.

## Node.js의 장점

- 자바스크립트를 사용하여 서버를 구축할 수 있기 때문에 클라이언트와 서버 부분을 한 언어로 개발할 수 있다(효율적)
- I/O작업이 많은 서버로 적합하다.

## Node.js의 단점

- 싱글 스레드이기 때문에 복잡한 연산 같이 CPU 작업이 많을 때는 부적합하다.
- 스레드가 멈추지 않도록 관리가 필요하다.
