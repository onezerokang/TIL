# 의식적인 연습과 학습 테스트

TDD와 리팩토링을 잘 하려면 연습을 많이 해야 한다.
하지만 단순히 연습만 해서는 실력이 늘지 않는다.
양치를 23년간 해왔지만 양치 실력이 초등학생 때보다 늘었는가?

연습을 많이 하더라도, 특별한 추가적인 연습을 하지 않으면 실력이 늘지 않는다.
연습도 효과적으로 해야 실력이 상승한다.

그렇다면 어떻게 하면 효과적으로 연습할 수 있는가?

목적의식 있는 연습에 얼마나 많은 시간을 투자했느냐가 중요하다.

이 차이로 10년 경력의 개발자보다 2~3년 경력의 개발자가 더 잘할 수도 있다.

## 의식적인 연습의 7가지 원칙

1. 효과적인 훈련 기법이 수립되어 있는 기술 연마
2. 개인의 **컴포트 존을 벗어난 지점에서 진행, 자신의 현재 능력을 살짝 넘어가는 작업을 지속적으로 시도**
3. **명확하고 구체적인 목표**를 가지고 진행(이번 연습을 할 때 무엇을 목표할 것인지를 정해두고 해야 한다)
4. 신중하고 계획적이다. 즉 개인이 온전히 집중하고 '의식적'으로 행동할 것을 요구
5. 피드백과 피드백에 따른 행동 변경을 수반
6. 효과적인 심정 표상을 만들어내는 한편으로 심적 표상에 의존
7. 기존에 습득한 기술의 특정 부분을 집중적으로 개선함으로써 발전시키고, 수정하는 과정을 수반

## 의식적인 연습 시작하기 - 장난감 프로젝트 찾기

회사 프로젝트로 TDD와 리팩토링을 연습하면 안된다.
연습하고 있는데 더 우선순위가 높은 업무가 떨어지면 연습을 중단해야 한다.

숙련되지 않은 기술을 회원이 사용하는 서비스에 바로 적용하는 것은 프로의 자세가 아니다. 연주자가 연습을 하지 않고 새로운 곡을 공연장에서 공연하는 것과 같다.

장난감 프로젝트로 시작해서, 삽질을 많이 해보는 것이 좋다.

## 학습 테스트

학습테스트란 내가 만든 API나 클래스를 테스트하는 것이 아닌 다른 사람이 만들어 놓은 라이브러리가 어떻게 동작하는지 단위 테스트를 만들어서 테스트하는 것이다.

새로운 기술을 배울 떄 애플리케이션을 개발하고 테스트하면 많은 시간이 소요된다. 하지만 학습 테스트는 단위 테스트 하나만 만들어도 바로 테스트가 가능하기 때문에 학습 속도가 엄청 빠르다.

예시: 학습 테스트로 스프링의 String을 공부하기

```java
public class StringTest {
    @Test
    public void split() {
        String[] values = "1".split(",");
        assertThat(values).contains("1");
        values = "1,2".split(",");
        assertThat(values).containsExactly("1", "2");
    }

    @Test
    public void substring() {
        String input = "(1,2)";
        String result = input.substring(1, input.length() - 1);
        assertThat(result).isEqualTo("1,2");
    }
}
```

학습테스트가 끝나면 내가 만든 메서드에 대한 테스트를 해본다. 이게 익숙해지면 TDD로 넘어간다.

### 학습 테스트의 연습 효과

- 단위테스트 방법을 학습할 수 있다.
- 단위테스트 도구의 사용법을 익힐 수 있다.
- 사용하는 API에 대한 학습 효과가 있다.

### 의식적인 연습에 대한 추가 설명

의식적인 연습이란, 연습을 할 때 구체적인 목표를 정하고 연습하는 것을 말한다.

예시: 이번 연습할 때는 else 예약어를 쓰지 않고 코드를 짜보겠다!
