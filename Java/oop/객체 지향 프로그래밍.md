# 객체 지향 프로그래밍

## 개요

객체 지향 프로그래밍은 프로그램을 객체의 집합으로 보는 패러다임이다. 각 객체는 상태와 행동을 갖고 서로 상호작용하여 프로그램이 동작하도록 한다.

## 등장 배경

객체 지향 프로그래밍 탄생 이전, 절차적/구조적 프로그래밍은 데이터와 함수가 분리된 형태로 프로그램을 작성했다.

객체 지향에서는 인간이 실제 사물(객체)를 인지하고 사고하는 방식대로 객체 단위의 프로그래밍이 가능하다.

## 객체 지향 프로그래밍의 특성

### 추상화

> 추상: 여러가지 사물이나 개념에서 공통되는 특성이나 속성 따위를 파악하는 작용

추상화는 사물이 갖고 있는 특성과 사물이 할 수 있는 행동 중 공통되거나 필요한 부분만 추출하여 모델링하는 것을 말한다.

예를 들어 현실 세계의 '사람'은 많은 특성과 행동을 가지고 있지만, 회원을 다루는 클래스를 만들 때는 이메일, 비밀번호, 이름 같이 필요한 정보만 뽑아서 클래스로 추상화할 수 있다.

따라서 추상화는 일종의 모델링이다.

### 상속

객체 지향 프로그래밍에서 상속은 상위 클래스의 데이터와 메소드를 하위 클래스에서 재사용하는 것을 말한다.

일반적으로 상속 관계는 부모-자식 관계로 알려져있지만, 재사용+확장 개념으로 보는 것이 맞다. 상속 관계는 "하위 클래스 is a kind of 상위 클래스" 문장을 만족해야 한다.

- **is a kind of 예시**:
  - 사람 is a kind of 동물: 사람은 일종의 동물이다.
  - 자동차 is a kind of 이동수단: 자동차는 일종의 이동수단이다.

### 캡슐화

캡슐화는 데이터와 메소드를 클래스로 묶는 것을 의미한다.

캡슐화된 데이터와 메소드는 접근 제어자와 getter/setter를 이용해 정보 은닉(information hiding)할 수 있다. 이를 통해 캡슐화 된 클래스를 사용할 때, 해당 클래스가 허용하는 데이터와 메소드에만 접근하게 하여 예상하지 못한 사이드 이펙트를 방지할 수 있다.

### 다형성

> 다형성: 같은 종(種)의 생물이면서도 어떤 형태나 형질이 다양하게 나타나는 현상. 예를 들면 암수에 따라 크기, 형태, 색깔 따위가 차이 나는 것이다.

객체 지향 프로그래밍에서 다형성이란 같은 타입의 클래스거나, 같은 이름의 메소드라도 사용되는 상황에 따라 다르게 해석되거나 실행되는 특성을 의미한다.

다형성의 종류는 다음과 같다.

- **자료형 다형성**:

  - 상위 클래스 타입의 참조 변수가 하위 클래스의 인스턴스를 참조할 수 있다.

  ```java
  class Animal {}

  class Cat extends Animal {}

  class Dog extends Animal {}

  class Cow extends Animal {}

  // 자료 다형성을 활용하여 부모 클래스로 자식 클래스를 하나의 리스트에 저장하였다.
  List<Animal> list = ArrayList<>();
  list.add(new cat());
  list.add(new dog());
  list.add(new cow());
  ```

- **매개변수 다형성**:
  - 변수의 타입뿐만 아니라 인터페이스나 매개변수에도 적용된다.
- **메소드 다형성**:
  - 오버로딩(Overloadindg): 같은 이름의 메소드지만 매개변수의 타입과 순서에 따라 다르게 실행된다.
  - 오버라이딩(Overriding): 같은 이름의 메소드지만 하위 클래스에서 재정의 할 경우 다르게 실행된다.
