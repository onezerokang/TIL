# 객체 지향 프로그래밍

## 1. 객체지향 프로그래밍이란?

객체지향 프로그래밍이란 프로그램을 데이터(속성)와 데이터를 처리하는 행동(메소드)을 가진 객체의 집합으로 보는 프로그래밍 패러다임이다. 프로그래머는 클래스를 통해 객체가 가져야 할 상태와 행동을 정의하고, 객체를 생성하고 조립하여 프로그램을 개발한다.

하나의 역할을 수행하기 위해 필요한 데이터와 메소드를 객체로 묶어 사용하기 때문에 재사용이 용이하고, 유연하며, 유지보수하기 쉽기 때문에 대규모 소프트웨어 개발에 사용된다.

## 2. 객체지향 프로그래밍의 등장 배경

프로그래밍 패러다임은 절차지향 프로그래밍 → 구조적 프로그래밍 → 객체지향 프로그래밍 순서대로 등장하였다. 각 패러다임은 이전 패러다임의 단점을 보완하기 위해 고안되었으며 이들의 특징은 다음과 같다.

- **절차지향 프로그래밍**: 입력된 데이터를 순차적으로 처리하며 결과를 도출한다. 초기 프로그래밍 언어에서 널리 사용되었으며, 대표적으로 C언어가 있다.
- **구조적 프로그래밍**: 절차지향 프로그래밍의 개념을 확장한 방식으로, 데이터를 순차적으로 처리하는 부분을 함수로 묶었다.
- **객체지향 프로그래밍**: 데이터와 데이터를 처리하는 함수를 객체로 묶었다. 재사용성, 유지보수성, 가독성이 크게 향상하였다. Java, Python, C++등이 객체지향 프로그래밍 언어의 예다.

즉 객체지향 프로그래밍은 데이터와 데이터를 처리하는 함수를 객체라는 단위로 묶어, 이전 패러다임보다 재사용성, 유지보수성, 가독성을 향상시킨 패러다임이다.

## 객체 지향 프로그래밍의 특성

### 추상화

> 추상: 여러가지 사물이나 개념에서 공통되는 특성이나 속성 따위를 파악하는 작용

추상화는 사물이 갖고 있는 특성과 사물이 할 수 있는 행동 중 공통되거나 필요한 부분만 추출하여 모델링하는 것을 말한다.

예를 들어 현실 세계의 '사람'은 많은 특성과 행동을 가지고 있지만, 회원을 다루는 클래스를 만들 때는 이메일, 비밀번호, 이름 같이 필요한 정보만 뽑아서 클래스로 추상화할 수 있다.

따라서 추상화는 일종의 모델링이다.

---

추상화는 사물을 사실적으로 재현하는 것이 아닌, 점, 선, 면, 색체 등의 단순한 표현을 이용해 그림을 그리는 것을 말한다.

컴퓨터 과학에서 추상화는 복잡한 자료, 모듈, 시스템으로부터 핵심적인 개념 또는 기능을 간추리는 것을 말한다.

프로그래밍에서 추상화는 클래스를 정의할 때 불필요한 부분을 생략하고, 객체의 속성 중 중요한 것에 중점을 두어 모델링하는 것을 말한다.

즉 클래스들의 중요하고 공통된 성질들을 추출하여 슈퍼 클래스를 선정하는 개념과, 이벤트 발생의 정확한 절차나 방법을 정의하지 않고 대표할 수 있는 표현으로 대체하는 것을 말한다.

객체 지향 프로그래밍의 추상화는 크게 두 가지로 나뉘는데 객체의 관련 속성만 표시하는 데이터 추상화와, 불필요한 세부 정보를 숨기는 제어 추상화가 있다.

### 상속

객체 지향 프로그래밍에서 상속은 상위 클래스의 데이터와 메소드를 하위 클래스에서 재사용하는 것을 말한다.

일반적으로 상속 관계는 부모-자식 관계로 알려져있지만, 재사용+확장 개념으로 보는 것이 맞다. 상속 관계는 "하위 클래스 is a kind of 상위 클래스" 문장을 만족해야 한다.

- **is a kind of 예시**:
  - 사람 is a kind of 동물: 사람은 일종의 동물이다.
  - 자동차 is a kind of 이동수단: 자동차는 일종의 이동수단이다.

### 캡슐화

캡슐화는 데이터와 메소드를 클래스로 묶는 것을 의미한다.

캡슐화된 데이터와 메소드는 접근 제어자와 getter/setter를 이용해 정보 은닉(information hiding)할 수 있다. 이를 통해 캡슐화 된 클래스를 사용할 때, 해당 클래스가 허용하는 데이터와 메소드에만 접근하게 하여 예상하지 못한 사이드 이펙트를 방지할 수 있다.

### 다형성

> 다형성: 같은 종(種)의 생물이면서도 어떤 형태나 형질이 다양하게 나타나는 현상. 예를 들면 암수에 따라 크기, 형태, 색깔 따위가 차이 나는 것이다.

객체 지향 프로그래밍에서 다형성이란 같은 타입의 클래스거나, 같은 이름의 메소드라도 사용되는 상황에 따라 다르게 해석되거나 실행되는 특성을 의미한다.

다형성의 종류는 다음과 같다.

- **자료형 다형성**:

  - 상위 클래스 타입의 참조 변수가 하위 클래스의 인스턴스를 참조할 수 있다.

  ```java
  class Animal {}

  class Cat extends Animal {}

  class Dog extends Animal {}

  class Cow extends Animal {}

  // 자료 다형성을 활용하여 부모 클래스로 자식 클래스를 하나의 리스트에 저장하였다.
  List<Animal> list = ArrayList<>();
  list.add(new cat());
  list.add(new dog());
  list.add(new cow());
  ```

- **매개변수 다형성**:
  - 변수의 타입뿐만 아니라 인터페이스나 매개변수에도 적용된다.
- **메소드 다형성**:
  - 오버로딩(Overloadindg): 같은 이름의 메소드지만 매개변수의 타입과 순서에 따라 다르게 실행된다.
  - 오버라이딩(Overriding): 같은 이름의 메소드지만 하위 클래스에서 재정의 할 경우 다르게 실행된다.
