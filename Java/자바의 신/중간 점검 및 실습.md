# 중간 점검 및 실습

## 1. 참조 자료형과 기본 자료형의 차이를 정리해주세요.

참조 자료형은 객체, 기본 자료형은 객체가 아닌 자료다.

두 자료형은 메소드의 매개변수로 넘겨질 때 차이점이 있다.

기본 자료형을 매개 변수로 넘기면 메소드 안에서는 원본 변수와 다른 변수를 복사하여 사용한다.
즉 메소드 안에서 매개변수를 변경하여도 원본 변수에는 영향을 주지 않는다.

참조 자료형의 경우 객체의 주소값을 복사하여 전달한다. 따라서 객체의 속성을 변경할 경우 원본 객체의 값이 변경된다.

## 2. 기본 자료형 8가지를 나열하고 각 타입의 특징을 정리해주세요.

- byte: -2<sup>7</sup> ~ 2<sup>7</sup> -1, 8비트
- short: -2<sup>15</sup> ~ 2<sup>15</sup> -1, 16비트
- int: -2<sup>31</sup> ~ 2<sup>31</sup> -1, 32비트
- long: -2<sup>63</sup> ~ 2<sup>63</sup> -1, 64비트
- char: 0 ~ 2<sup>16</sup> -1, 16비트
- boolean: true와 false의 값이 있다.
- float: 32비트 단일 정확도 소수
- double: 64비트 이중 정확도 소수

## 4. if 문이 용도는 무엇이며 if-else와 if-else if는 어떤 점이 다른지 정리해주세요.

if 문은 조건에 맞는 로직을 실행 시키는 조건문이다.
if-else는 이분법적으로 분기 처리를 하지만
if-else if는 다수의 조건으로 분기 처리할 수 있다.

## 5. switch-case문의 용도를 정리해 주세요.

switch-case문은 주어진 값이 case와 일치할 경우 해당 로직을 실행시키는 용도이다.
if문과 다르게 일치하는 case가 있더라도 중단되지 않으니 필요에 따라 break를 사용해야 한다.

## 8. 생성자는 무엇을 하는 데 사용하는 것이며, 별도로 만들지 않아도 자동으로 생성되는 생성자에 대해서 정리해 주세요.

생성자는 클래스에서 인스턴스를 생성할 때 사용한다. 메소드와 비슷한 구조이지만 리턴타입이 없고 생성자의 이름은 클래스명과 같다.

기본 생성자의 경우 어떤 생성자도 없을 경우 컴파일 단계에서 만들어주는 매개 변수가 없는 생성자다.

단, 다른 생성자가 한 개라도 있을 경우 만들어지지 않기 때문에 이때는 명시적으로 선언해줘야 한다.

## 9. Overloading은 무엇인가요? public void setData(int a)라는 메소드를 원하는 대로 Overloading 해주세요.

오버로딩이란 메소드명은 같지만 매개변수를 다르게 하여 같은 메소드명으로 여러 데이터 타입을 처리할 수 있게 하는 방법이다.

```java
public class Overloading {
    public void setData(int a) {}

    public void setData(String a) {}
}
```

## 13. 인터페이스, abstract 클래스, Enum 클래스가 있는데 각각의 특징 및 다른 점을 정리해 주세요.

인터페이스는 추상메소드를 갖는다. 자바의 클래스가 인터페이스를 implements 키워드로 확장하면 추상메소드를 구현해야 한다.

abstract 클래스는 한 개 이상의 추상 메소드를 갖는 클래스다. 인터페이스와 다르게 실제로 구현된 메소드를 갖고 있을 수 있고 extends 키워드로 상속한다.

Enum 클래스는 상수의 집합을 모아 놓은 클래스이며, 각 상수는 Enum 클래스의 인스턴스이다.
이는 싱글톤 패턴으로 생성되어, 프로그램 전체에서 사용된다.

## 16. 변수를 final로 선언하는 것이 어떤 의미가 있는지 정리해 주세요.

final은 변수를 상수로 선언할 때 사용한다. 해당 변수는 선언과 동시에 반드시 초기화 되어야 한다.

## 17. 클래스를 final로 선언하는 것이 어떤 의미가 있는지 정리해 주세요.

클래스를 final로 선언하면 다른 클래스가 해당 클래스를 상속 받을 수 없게 된다. 예를 들어 String 클래스는 final로 선언되어있다.

## 18. 변수를 static으로 선언하는 것이 어떤 의미가 있는지 정리해 주세요.

변수를 static으로 선언하면 해당 변수는 클래스 변수가 된다.

클래스 변수는 인스턴스를 생성하지 않고 클래스에서 바로 접근할 수 있는 변수다.

해당 클래스를 사용하는 인스턴스들은 모두 같은 클래스 변수를 바라보고 있기 때문에 값이 변경되어 예상하지 못한 사이드 이펙트가 발생되지 않게 해야 한다.

## 19. 메소드를 static으로 선언하는 것이 어떤 의미가 있는지 정리해 주세요.

메소드를 static으로 선언할 경우 해당 변수는 인스턴스를 생성하지 않고 호출할 수 있는 메소드가 된다.

static 메소드의 경우 인스턴스 변수에 접근할 수 없으며 static 변수에만 접근할 수 있다.

## 20. try-catch-finally 블록은 왜 사용하고 각각의 블록이 어떤 용도로 사용되는지 정리해 주세요.

try-catch-finally 블록은 예외를 처리하기 위한 블록이다.

try 블록에 예외가 발생할 수 있는 로직을 넣고 예외가 발생할 경우 catch 블록에서 처리한다.

finally 블록은 예외 발생 여부와 상관 없이 무조건 실행되는 블록이다.
