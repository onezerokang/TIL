# 3장 자바를 제대로 알려면 객체가 무엇인지를 알아야 해요

자바는 객체지향 언어.

세상의 모든 사물들은 객체로 되어 있다.

클래스는 상태와 행위가 있다.

즉 자바는 세상의 모든 사물이나 개념을 클래스로 표현할 수 있는 언어

```java
package chap3;

public class Car {
    // 자동차의 상태
    int speed;
    int distance;
    String color;

    // 생성자: 객체를 생성할 때 필요하다.
    public Car() {

    }

    // 자동차의 행위
    public void speedUp() {
        speed += 5;

    }

    public void breakDown() {
        speed -= 10;
    }

    public int getCurrentSpeed() {
        return speed;
    }

}

```

## 클래스와 객체는 구분해야 한다.

실제 사물을 나타내기 위한 것을 객체 혹은 인스턴스라고 한다.

```java
package chap3;

public class CarManager {
    public static void main(String[] args) {
        Car dogCar = new Car();
        Car cowCar = new Car();
    }
}
```

Car()라고 하는 메소드 같이 생긴 생성자를 사용한다.
생성자는 객체를 생성하기 위한 거의 유일한 도구.
매개 변수가 없는 생성자를 기본 생성자라고 하는데, 이는 클래스를 컴파일할 때 클래프 파일안에 자동으로 생성되기에
직접 작성하지 않아도 된다.

클래스는 new 예약어와 생성자를 호출하여 객체를 만들어야 한다.
클래스 자체로는 일을 할 수 없고, 객체를 생성해야만 일을 시킬 수 있다

?그냥 자바스크립트처럼 바로 클래스 없이 함수를 호출하게 하면 될텐데, 왜 클래스를 사용하는 것일까?

```java
package chap3;

public class CarManager {
    public static void main(String[] args) {
        Car dogCar = new Car();
        dogCar.speedUp();
        dogCar.speedUp();
        System.out.println(dogCar.getCurrentSpeed());
        dogCar.breakDown();
        System.out.println(dogCar.getCurrentSpeed());
    }
}

```
