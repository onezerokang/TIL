# System.out.println()대신 로거(Logger)를 사용하자

System.out.println()은 편하게 값을 출력할 수 있다. 많은 개발자가 예외가 발생했을 때 예외 내용을 출력하거나, 디버깅하면서 변수에 저장된 값을 확인하기 위해 사용한다. 하지만 System.out.println()은 실무에서는 사용하면 안되는 메소드다. 로그를 출력할 때는 로거(Logger)를 사용해야 하고, 디버깅할 때는 디버거를 사용하자.

System.out.println()을 사용하면 안되는 이유는 다음과 같다.

1. 애플리케이션 성능 저하
2. 휘발
3. 가독성
4. 로그 레벨 설정 불가

## 1. 성능 저하

System.out.println() 메소드의 내부는 synchronized 키워드로 구현되어있다. 이 메소드는

로그를 콘솔에 출력하거나, 파일에 저장하는 작업은 처리속도가 느린 I/O 작업이다.

운영체제는 IO작업을 처리하는 동안 이를 기다리지 않고 다른 스레드의 작업을 한다.

CPU가 놀고 있으면 작업 효율이 떨어진다.

하지만 synchronized 키워드를 사용하면 해당 작업이 끝날 때까지 다른 스레드로 스위칭할 수 없다.

실제로 성능을 측정해보자.

## 3. 로그 레벨 설정 불가

일반적인 로거 프레임워크는 로깅 레벨을 설정할 수 있다. 이를 통해 로컬 개발 환경, 프로덕션 환경 등 각 환경에 맞는 로그를 출력하고 저장하는 등의 처리가 가능하다. 하지만 System.out.println()은 이런 기능을 제공하지 않는다.

다음은 Log4j에서 로그 레벨을 지정한 예시다.

logger.error();
logger.warn();
logger.info();
logger.debug();
logger.trace();

이렇게 하고 개발환경에서는 debug 레벨까지 출력하게 하고, 프로덕션 환경에서는 error 레벨만 출력하게 하는 등의 설정이 가능하다. 프로덕션 환경에서 디버그 용 로그를 출력할 필요가 없기 때문에 유의미한 데이터만 출력 + 성능면에서도 이득이다.
