# 컴파일 언어 vs 인터프리터 언어

자바는 컴파일 언어의 특징과 인터프리터 언어의 특징을 모두 갖는다. 이 이유를 알기 전 컴파일 언어와 인터프리터 언어가 무엇인지 알아보자.

- **컴파일 언어**:
  - **정의**: 컴파일 언어는 소스 코드를 모두 기계어로 변환하는 과정을 거치는 언어다. 이 변환작업은 컴파일러에 의해 수행된다.
  - **장점**: 전체 코드를 컴파일한 후 실행하기 때문에 실행 속도가 빠르다. 컴파일 과정에서 오류를 미리 발견할 수 있다.
  - **단점**: 코드가 변경 시 매번 컴파일 과정을 거쳐야 한다. 또한 생성된 실행파일은 대부분의 경우 특정 플랫폼에 종속적이다.
- **인터프리터 언어**: 인터프리터 언어는 한줄씩 기계어로 변환하고 실행하는 언어다. 이 작업은 인터프리터에 의해 수행된다.
  - **장점**: 코드를 컴파일 과정 없이 바로 실행할 수 있어 개발과 디버깅에 용이하다.
  - **단점**: 한 줄씩 기계어로 번역하고 실행하기 때문에 컴파일 언어보다 실행 속도가 느리다.

그렇다면 자바는 왜 컴파일 언어와 인터프리터 언어의 특징을 모두 갖는 것일까?

자바가 컴파일 되어서 실행되는 과정은 다음과 같다.

1. 자바 컴파일러는 자바 소스 파일(.java 파일)을 JVM이 읽을 수 있는 바이트 코드로 컴파일 한다.
2. JVM은 운영체제나 CPU 아키텍처에 맞게 바이트 코드를 기계어로 변환하며 이때 인터프리터 방식 또는 JIT(Just-In-Time) 컴파일 방식을 사용한다.

자바는 플랫폼에 종속되지 않기 위해 소스 코드를 JVM이 읽을 수 있는 바이트 코드로 변환하고, 이를 인터프리터 방식 또는 JIT 컴파일 방식으로 실행하는 과정을 거친다. 따라서 자바는 컴파일 언어와 인터프리터 언어의 특징을 모두 갖는다고 볼 수 있다.
