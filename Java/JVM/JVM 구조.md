# JVM 구조

- 클래스 로더 시스템: 컴파일된 바이트 코드를 읽어들여서 메모리에 적절하게 배치하는게 클래스 로더가 하는 일이다. 다음은 클래스 로더가 하는 일이다.
  - 로딩: 클래스 파일에서 바이트 코드 읽어오기
  - 링크: 레퍼런스 연결
  - 초기화: 클래스의 static한 값 초기화
- 메모리:
  - 스레드에 국한되는 영역(스레드마다 하나씩 존재):
    - 스택: method call시 스택 프레임을 쌓는다.
    - PC: 실행할 스택 프레임을 가리키는 포인터
    - 네이티브 메소드 스택: 네이티브 메소드를 사용할 때 사용되는 별도의 스택(네이티브 메서드는 C/C++로 구현된 메서드. 예시: currentThread())
  - 힙: 실제 인스턴스들을 저장한다.
  - 메소드: 클래스 수준의 정보를 저장한다. 클래스 이름, 풀패키지 경로, 부모 클래스 이름, 변수, static 변수 등. 공유하여 다른 영역에서 사용 가능
- 네이티브 메소드 인터페이스, 네이티브 메소드 라이브러리:
  - native method를 사용할 수 있는 메소드와 실제 구현체
- 실행 엔진
  - 인터프리터: 바이트 코드를 한줄 씩 네이티브 언어로 변환하고 실행. 똑같은 코드를 매번 한줄씩 변환하는 것은 비효율적. 반복적인 코드는 JIT 컴파일러로 보낸다.
  - JIT 컴파일러: 반복된 코드를 전부 미리 바꿔둔다. 해당 라인에 걸렸을 때 인터프리터가 변환하는 것이 아닌 네이티브 코드를 실행한다.
  - GC: 가비지 컬렉터
    - stop the world: gc 발생 시 멈춤현상을 최소화 하는 gc
    - 더이상 쓰이지 않는 객체를 찾아서 정리하는 역할

정리

- 클래스 로더가 읽어서 메모리에 배치한다.
- 스레드가 만들어지면 스택, pc 등이 생성된다.
- 실행할 때 바이트코드를 한줄씩 실행한다. 반복되는 코드는 JIT 컴파일러가 바이트코드로 변환한다. 메모리 최적화를 위해 참조하지 않는 인스턴스를 GC가 정리한다.
- 네이티브 라이브러리를 사용할 경우 항상 JNI를 통해서 사용한다.
