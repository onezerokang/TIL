# 재귀

재귀는 완전 탐색, 이진 탐색, 동적 프로그래밍 등 다양한 알고리즘에 사용되는 기본 개념이기에, 재귀적 사고에 익숙해져야 한다.

## 재귀란?

재귀는 메서드 내에서 자기 자신을 호출하도록 하여 반복적인 개념을 구현하는 것이다.

하나의 재귀 호출은 하나의 부분 문제를 해결하며, 부분 문제를 푸는 재귀 호출에서 또 다른 부분 문제를 푸는 재귀 호출을 수행한다. 이렇게 부분 문제들의 답이 모여 상위 호출에 대한 답을 구성한다.

주의: 재귀 호출이 종료되지 않고 너무 깊게 들어가면 StackOverFlowError 예외가 발생한다. 재귀 호출의 깊이를 10,000 ~ 20,000 이하로 유지 시켜야 한다.

## 재귀 정의하기

예시 문제를 재귀로 풀어보자.

- 문제: 0 이상의 두 정수 n, m이 주어졌을 때 n<sup>m</sup>을 구하라

### 상태 정의하기

재귀는 부분 문제를 해결해 나가는 풀이 방법이다. 부분 문제는 각각 하나의 명확한 문제를 나타내어 하나의 답을 낼 수 있어야 한다. 답을 내는 데 입력되는 변수들이 필요하고, 이렇게 답을 결정하는 변수를 **상태**라고 한다. 그리고 부분문제는 하나의 상태에 대한 답을 찾는 문제가 된다.

위 문제에서 상태는 (n, m)으로 정의할 수 있다.

### 종료 조건

재귀는 하나의 문제를 부분 문제로 나누거나 부분 문제의 답을 이용하여 원본 문제의 답을 풀 수 있어야 한다. 그리고 부분 문제들을 생성하다보면 언젠가는 종료되어야 한다. 이렇게 즉시 답이 나오는 상태를 검사하여 답을 반환할 수 있도록 하는 것을 종료 조건이라고 한다.

- (n, 0) = 1
- (0 m) = 1
- (1, m) = 1

### 점화식

가장 큰 상태가 어떤 과정을 거쳐 가장 작은 상태인 종료 조건에 도달하는지를 정의해야 한다. 부분 문제는 같은 규칙으로 더 작은 부분 문제로 진행되어야 한다. 따라서 상태 또한 하나의 규칙으로 더 작은 상태로 전이되어야 한다. 이렇게 상태를 전이시키는 규칙을 점화식이라고 한다.

- n<sup>m</sup> = n \* n<sup>m - 1</sup>
- (n, m) = n \* (n, m - 1)

## 코드 작성하기

- 재귀 문제의 상태, 종료 조건(가장 작은 상태), 점화식을 세웠다면 이를 코드로 옮겨야 한다.

### 코드 변환하기

- 상태: (n, m): n<sup>m</sup>
- 종료 조건:
  - (n, 0) = 1
  - (0, m) = 1
  - (1, m) = 1
- 점화식:
  - (n, m) = n \* (n, m - 1)

#### 부분 문제 나타내기

재귀 호출 구현에서 하나의 부분 문제는 하나의 메서드가 담당한다. 예시 문제에서는 상태 (n, m)을 나타내는 부분 문제를 풀어야 한다.

```java
private int power(int n, int m) {
    // 종료 조건, 점화식 구현하기
}
```

#### 종료 조건 작성하기

종료 조건은 조건을 만족할 경우 그 즉시 답을 낼 수 있다.

```java
private int power(int n, int m) {
    if (n == 0) return 1;
    if (n == 1) return 1;
    if (m == 0) return 1;
    // 점화식 구현하기
}
```

### 점화식 구현하기

점화식은 하나의 부분 문제를 더 작은 상태를 나타내는 부분 문제를 이용해 푸는 규칙이다. 부분 문제를 푸는 메서드는 power()이므로, 이 메서드에서 더 작은 상태를 넣어 주면 해당 상태를 풀 수 있다.

```java
private int power(int n, int m) {
    if (n == 0) return 1;
    if (n == 1) return 1;
    if (m == 0) return 1;

    return n * power(n, m - 1);
}
```
